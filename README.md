# Integracion de patrones
Repositorio para la aplicación de multiples patrones dirigidos a una empresa de entrega de domicilios

## Paquetes
Las clases están separadas según los patrones que se deben implementar de la siguiente forma:

- [Decorator](https://github.com/xDylanSilva/Integracion-de-patrones/tree/main/packagedelivery/src/main/java/delivery/decorator)
- [Factory](https://github.com/xDylanSilva/Integracion-de-patrones/tree/main/packagedelivery/src/main/java/delivery/factory)
- [Observer](https://github.com/xDylanSilva/Integracion-de-patrones/tree/main/packagedelivery/src/main/java/delivery/observer)
- [Singleton](https://github.com/xDylanSilva/Integracion-de-patrones/tree/main/packagedelivery/src/main/java/delivery/singleton)
- [Strategy](https://github.com/xDylanSilva/Integracion-de-patrones/tree/main/packagedelivery/src/main/java/delivery/strategy)

Además de esto el UML se encuentra al inicio del proyecto y la clase [Main](https://github.com/xDylanSilva/Integracion-de-patrones/blob/main/packagedelivery/src/main/java/delivery/Main.java) está en src/main/java/delivery sin ninguna carpeta para que sea fácil de disstinguir.
